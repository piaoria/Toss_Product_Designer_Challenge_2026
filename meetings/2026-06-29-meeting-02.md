## 1단계: 본질적 문제 정의

### 문제 1: "의사결정 주체의 분산으로 인한 책임 회피 구조"
**한 줄 요약:** 누가 최종 결정을 하는가 불명확하여, 모두가 제안하고 아무도 책임지지 않는 상황 발생

#### 심리적·행동적 분석
- **현상:** 카카오톡/이메일에서 A가 "목요일 3시 어때요?"라 제안 → B, C가 "좋아요" → D가 "저는 목요일 외근..." → E가 조용히 시간을 확인 → F는 반응 없음
- **본질 문제:** "좋아요"의 의미가 불명확함
  - 긍정: "나는 가능해"인가, 아니면 "그거 좋은 아이디어네"인가?
  - 침묵: "동의"인가, "아직 확인 중"인가, "반대인데 말 안 하는 건가"?
- **심리 기제:** *[사회적 합의 편향(social conformity bias)]*
  - 이미 2명이 좋다고 했으니 나도 따라 좋다고 함
  - 이미 의사가 정해진 것 같으니 반대 의견을 못 제시함
- **결과:** 확실성 없이 "일단 이 시간으로 하자" 식 약속 → 직전에 "어? 나 못 갔는데?"라는 갈등 발생

---

### 문제 2: "개인의 제약조건이 가시화되지 않아 발생하는 반복적 협상 비용"
**한 줄 요약:** 각자의 불가능한 시간을 몰라서, 같은 시간을 계속 제안했다 취소했다를 반복

#### 심리적·행동적 분석
- **현상:** 
  - "금요일은 어때?" (답변 대기) → "금요일 외근 많아서..." (1차 거절)
  - 다시 "그럼 목요일?" → "목요일 오후는 안 돼..." (2차 거절)
  - 누군가 "너 피할 수 없는 시간 미리 말 안 해?" (감정 소비)

- **본질 문제:** *정보 비대칭성(Information Asymmetry)*
  - 각자의 제약을 개별적으로 알려야 함 (일일이 대답)
  - 한 사람의 불가능한 시간이 전체 협상 구조를 바꿈 (특히 필수 참석자)
  
- **심리 기제:** *[의사결정 피로(Decision Fatigue)]*
  - 같은 질문을 3번 받으면 답변 성의가 떨어짐
  - "나 때문에 자꾸 불편하게 하는 건가"라는 죄책감 + 짜증 동시 발생
  
- **행동 결과:**
  - 정확한 답변 대신 "아 뭐... 가능하면 가겠습니다" 같은 모호한 응답
  - 점심 직후 기피, 월요일 외근 많음 같은 **정성적 정보**는 전달하지만, 이를 체계적으로 처리할 방법이 없음

---

### 문제 3: "필수/선택 참석자 구분이 무의미해지는 논리적 붕괴"
**한 줄 요약:** 필수 참석자 1명의 불가능한 시간에 전체가 발목 잡혀, 결국 선택자들의 참석 여부가 불명확해짐

#### 심리적·행동적 분석
- **현상:**
  - 필수 참석자 A: "월·수·금은 외근, 점심 직후 불가" 
  - 선택 참석자 B~F: "어? 그럼 화·목 오전/오후인데... 그 중에 다 가능한 시간이 있나?"
  - 결과: 어떤 시간을 잡아도 누군가는 "어차피 선택이니 저는 패스할게요" 식으로 이탈

- **본질 문제:** *[우선순위의 모호함으로 인한 신뢰도 붕괴]*
  - "필수"는 절대 불가능한 일정은 피해야 하지만, **선택은 최선의 노력**이어야 함
  - 그런데 현재 방식은 선택자도 필수자와 동일한 제약을 받아야 하므로, 선택자가 "굳이 이렇게까지?" 이탈
  - 필수자의 불편함을 선택자에게 押し付け(강요)하는 구조

- **심리 기제:** *[공정성 위반(Violation of Fairness)]*
  - 최우선순위 참석자의 불가능한 시간이 전체를 제약 → 모두가 손해
  - 선택자는 "내 시간도 소중한데?" 느낌
  
- **행동 결과:**
  - "아 뭐 어차피 선택이고, 이 회의 없어도 되긴 하고..." → **무임승차(free-riding)**
  - 회의 직전 취소자 발생 (당일 "죄송한데 개인 사정이...")

## 1단계: Problem Framer 통합 분석

Problem Framer가 정의한 3가지 핵심 문제를 UX 관점에서 재해석:

| 문제 | 근본 원인 | UX 솔루션 방향 |
|------|---------|-------------|
| **의사결정 주체 분산** | 모호한 응답 상태("좋아요"의 다의성) | 명시적 상태 표현(Yes/No/Maybe/Pending) |
| **개인 제약 조건 비가시화** | 일일이 대답하는 반복 협상 | 일괄 입력 후 자동 필터링 |
| **필수/선택 가중치 붕괴** | 우선순위가 실제 의사결정에 미반영 | 시각적 가중치 + 논리적 필터링 |

이를 해결하는 설계 철학:
> **"한 번의 입력으로, 모두가 같은 그림을 보게 하라"**

짚어주는 역할이에요.

## [페르소나 1] 주최자 — 책임감의 무게를 느낀 사람

**첫인상:**
"오, 이거 뭔가 다르네. 일반 캘린더 앱이랑 달라서... 혼자만 잘 이해하면 되지 않을까?" 
화면을 열자마자 [화면 1] 기본 정보 입력이 나온다. "회의명, 기간, 소요시간, 참석자 선택"... 
이 정도면 명확하다. 필수/선택을 내가 직접 지정하는 거니까, 책임은 내가 지는 느낌이 든다.
**불안감이 살짝 인다.** "혹시 내가 누군가를 필수에서 선택으로 빼서 그 사람이 섭섭해하진 않을까?"

**막히는 지점:**
[화면 2]로 가서 칼렌더를 보는데... 
"어? 왜 특정 슬롯들이 회색이고, 이건 주황색 스트라이프고, 이건 보라 도트지?" 

**Visual Critic의 설명을 읽으면** 이해가 된다. 
- 빨강 = 불가능 (A의 외근)
- 주황 스트라이프 = 기피 (A의 점심 직후)
- 보라 도트 = 외근 많음 (B의 화요일)

**하지만 내가 처음 보는 입장에서는?** 
"왜 색깔이 이렇게 많아? 범례(Legend)가 있나?" 
칼렌더 상단이나 옆에 작은 범례 박스 "(빨강=불가, 주황=기피, ...)" 정도가 있어야 한다.

더 크리티컬한 점: **[화면 3] 자동 일정 매칭 제안**을 봤을 때,
"화요일 10:00~11:00"이 제안되는데, 
- "필수 전원 가능, 선택자 5/5"라고 나온다. 
- 그 아래 신뢰도 ★★★★★

**여기서 막힌다:**
"어? 선택자가 꼭 6명 중 5명을 가야 하나? 
내가 지정할 때는 선택자가 누구더라... 정확히 몇 명이었지?"

[화면 1]에서 내가 입력한 '참석자 구성'이 **[화면 3]의 제안 칼럼에 다시 보여야** 한다.
예: "필수 3명 (A, B, C) + 선택 3명 (D, E, F)"

또 다른 문제: **"이 제안을 '확정'했을 때 정확히 뭐가 일어나는가?"**
- 모두의 캘린더에 자동 등록된다고 했는데, 
- **진짜 필수자는 100% 가는 거고, 선택자는 "그냥 예정"인 건가?**
- 아니면 **선택자도 확정 후에 다시 한 번 의사 확인을 받는** 건가?

[화면 3]에 마이크로 카피(Microcopy)가 부족하다.
"확정하기" 버튼 옆에 작은 글씨로:
> "필수 참석자는 일정이 확정되고, 선택 참석자는 최종 확인 메시지를 받습니다"

이 정도면 내 책임감이 줄어들 것 같다.

**오해할 부분:**
1. **색상 범례의 부재** → "이 칼렌더 색깔이 뭔 뜻이지?" 첫 사용자는 10초 이상 헷갈림
2. **제안의 "신뢰도" 의미** → ★★★★★가 뭘 기준으로 한 평가인가? 
   - "모든 필수자 + 모든 선택자가 가능하면 만점"이라고 가정하는데, 
   - 그게 정말 **"이 시간대가 최고로 좋다"**는 뜻인가, 아니면 **"논리적으로 최적"**이라는 뜻인가?
3. **"다시 제안받기" 버튼의 의미** → 이미 모든 사람이 응답했는데, 뭘 다시 제안하라는 건가? 
   - 혹시 **필수자 하나가 불가능이라고 답변했을 때, 두 번째로 "다시 물어보기"**라는 뜻인가?

**감정 반응:**
- **초반(화면 1):** 만족감 70% + 책임감 30%
  - "오, 깔끔하게 정의하라는 건 좋은데... 혹시 내가 잘못 입력하면 모임이 망가지는 건 아닐까?"
  
- **중반(화면 2):** 혼란감 40% + 흥미로움 40% + 신뢰감 20%
  - "색깔도 많고, 패턴도 많은데... 근데 이게 자동으로 필수자 제약을 반영한 건가? 신기하네"
  
- **후반(화면 3):** 안도감 60% + 약간의 불안감 40%
  - "오 자동으로 최적 시간을 찾아줬네. 이거 하나 클릭하면 끝이구나. 그런데... 정말 모두가 이 시간대를 보고 동의하는 건가?"

**한 줄 제안:**
> **[화면 2]의 칼렌더 위에 작은 범례 바를 항상 보이게 두고, [화면 3]의 "확정하기" 버튼 옆에 "필수/선택 처리 방식" 설명 마이크로카피를 추가하세요.**
> 그래야 내가 최종 버튼을 누를 때 진짜 책임감 없이 누를 수 있습니다.
