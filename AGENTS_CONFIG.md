## 🤖 에이전트 구성

실행 순서대로 4개. 각 에이전트는 이전 에이전트의 결과를 **누적**으로 입력받습니다.

### 1. 🔍 Agent A — Problem Framer
- **role:** UX 리서처 / 문제 정의 전문가
- **컨텍스트 입력:** 과제 조건만 (이전 에이전트 없음)
- **임무:** 일정 조율의 본질적 문제 3가지 정의 + 심리적·행동적 근거 + 조건(필수/선택, 외근, 점심 후 기피)이 문제를 심화시키는 방식 분석

### 2. 🎨 Agent B — UX Designer
- **role:** 프로덕트 디자이너 / 인터랙션 설계자
- **컨텍스트 입력:** 과제 조건 + A 결과
- **임무:** Trigger→제안→응답→확정→알림 전체 플로우 + 핵심 화면 3개 + 가설 기반 설계 의도("~게 보여주면 ~게 인지하여 ~게 행동") + 필수/선택 가중치 시각화

### 3. ✏️ Agent C — Visual Critic
- **role:** UI 스페셜리스트 / 비주얼 디테일 검증자
- **컨텍스트 입력:** 과제 조건 + A + B 결과
- **임무:** 컬러/타이포/아이콘 시스템 제안, 4가지 상태(가용/불가/기피/외근) 컬러 구분(hex), 모바일/데스크톱 레이아웃 차이, 토스 영감 컴포넌트 디테일 2개 (실제 hex·px 수치 포함)

### 4. ⚖️ Agent D — Toss Evaluator
- **role:** 기획 검증자 / 토스 기준 평가자
- **컨텍스트 입력:** 과제 조건 + A + B + C 전체
- **임무:** 토스 3대 기준(문제정의/솔루션설계/조형완성도) Pass·Fail + 10점 만점 점수 + 보완 제안 + 최종 강점 2 / 개선 2

## 📥 공유 컨텍스트 (TASK_CONTEXT)

모든 에이전트가 공유하는 과제 조건:

- **핵심 과제:** 6명의 동료가 1시간 회의 일정을 잡는 경험 설계 (Web/App UI)
- **제약:** 동료 6명, 다음 주까지, 회의 시간 정확히 1시간
- **개인 선호:** 특정 인물 점심 직후 기피 / 특정 인물 특정 요일 외근 / 필수·선택 참석자 가중치
- **토스 평가 기준:** ① 본질적 문제 정의 ② 가설 기반 솔루션 설계 ③ 조형적 완성도와 디테일
- **토스 핵심 가치:** 복잡함의 단순화 / Full-Cycle 오너십 / 논리적 설득력

> 전체 시스템 프롬프트 원문은 `harness.mjs`의 `AGENTS` 객체와 `TASK_CONTEXT` 상수에 있습니다.
