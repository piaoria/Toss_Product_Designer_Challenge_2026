## 🤖 에이전트 구성

실행 순서대로 5개. 각 에이전트는 이전 에이전트의 결과를 **누적**으로 입력받습니다.

**모델 배정:** A·B·C·E = `claude-opus-4-8` (추론·설계·디자인 품질), D = `claude-haiku-4-5` (페르소나 QA, 저비용)
**플랫폼/산출물:** 모든 에이전트가 공유 컨텍스트로 "모바일 우선+반응형", "Figma Make 입력용 구체성"을 전제로 작업

### 1. 🔍 Agent A — Problem Framer
- **role:** UX 리서처 / 문제 정의 전문가
- **컨텍스트 입력:** 과제 조건만 (이전 에이전트 없음)
- **임무:** 일정 조율의 본질적 문제 3가지 정의 + 심리적·행동적 근거 + 조건(필수/선택, 외근, 점심 후 기피)이 문제를 심화시키는 방식 분석

### 2. 🎨 Agent B — UX Designer
- **role:** 프로덕트 디자이너 / 인터랙션 설계자
- **컨텍스트 입력:** 과제 조건 + A 결과
- **임무:** Trigger→제안→응답→확정→알림 전체 플로우 + 핵심 화면 3개 + 가설 기반 설계 의도("~게 보여주면 ~게 인지하여 ~게 행동") + 필수/선택 가중치 시각화

### 3. ✏️ Agent C — Visual Critic
- **role:** UI 스페셜리스트 / 비주얼 디테일 검증자
- **컨텍스트 입력:** 과제 조건 + A + B 결과
- **임무:** 컬러/타이포/아이콘 시스템 제안, 4가지 상태(가용/불가/기피/외근) 컬러 구분(hex), 모바일/데스크톱 레이아웃 차이, 토스 영감 컴포넌트 디테일 2개 (실제 hex·px 수치 포함)

### 4. 👥 Agent D — User QA Panel
- **role:** 실사용자 QA / 페르소나 검증자
- **컨텍스트 입력:** 과제 조건 + A + B + C
- **임무:** 한 에이전트가 과제 시나리오의 6명 동료 페르소나에 빙의해 B+C 설계 플로우를 1인칭으로 사용. 페르소나별 첫인상/막히는 지점/오해할 부분/감정 반응/한 줄 제안 + [공통 발견 사항] 핵심 사용성 이슈 2~3개
- **6 페르소나:** ①주최자 ②점심직후 기피자 ③외근 잦은 동료 ④선택 참석자 ⑤바쁜 필수참석자(리더) ⑥신규입사자/디지털 약자

### 5. ⚖️ Agent E — Toss Evaluator
- **role:** 기획 검증자 / 토스 기준 평가자
- **컨텍스트 입력:** 과제 조건 + A + B + C + D 전체
- **임무:** 토스 3대 기준(문제정의/솔루션설계/조형완성도) Pass·Fail + 10점 만점 + 보완 제안 + 최종 강점 2 / 개선 2. 추가 점검: ④모바일+반응형 고려 ⑤Figma Make 구현 가능성. **실사용자 QA(D) 이슈가 솔루션 가설을 뒷받침/반박하는지 반영**

## 📥 공유 컨텍스트 (TASK_CONTEXT)

모든 에이전트가 공유하는 과제 조건:

- **핵심 과제:** 6명의 동료가 1시간 회의 일정을 잡는 경험 설계 (Web/App UI)
- **제약:** 동료 6명, 다음 주까지, 회의 시간 정확히 1시간
- **개인 선호:** 특정 인물 점심 직후 기피 / 특정 인물 특정 요일 외근 / 필수·선택 참석자 가중치
- **토스 평가 기준:** ① 본질적 문제 정의 ② 가설 기반 솔루션 설계 ③ 조형적 완성도와 디테일
- **토스 핵심 가치:** 복잡함의 단순화 / Full-Cycle 오너십 / 논리적 설득력
- **플랫폼:** 모바일 우선 + 데스크톱 웹 반응형 (모바일/웹 레이아웃 차이를 항상 함께 명시)
- **최종 산출물:** Figma Make 입력용 — 기능은 동작 단위로, 화면은 컴포넌트+카피+상태까지, 디자인은 hex/폰트/간격/반경까지 구체화

> 전체 시스템 프롬프트 원문은 `harness.mjs`의 `AGENTS` 객체와 `TASK_CONTEXT` 상수에 있습니다.

- 실행 시 콘솔에 에이전트별 진행상황 + 토큰(입력/출력/합계) + 소요시간 출력

- **모든 회의록을 git으로 추적** — 진행 기록이므로 별도 gitignore 없음 (`meetings/` 폴더에 누적)
- 초기 마일스톤 회의록은 `meetings/2026-06-29-meeting-01.md`, `-02.md`로 보존
